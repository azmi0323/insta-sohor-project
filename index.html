অধিকাংশ মানুষ জাভাস্ক্রিপ্ট সম্পর্কে যা জানতে চায়?

জাভাস্ক্রিপ্ট শিখতে দুটি গুরুত্বপূর্ণ অংশ রয়েছে: ওয়েব ব্রাউজার কীভাবে কাজ করে এবং ডকুমেন্ট অবজেক্ট মডেল (DOM) কীভাবে কাজ। প্রথমে ওয়েব পেজ তৈরি করার জন্য HTML এবং CSS কে DOM দ্বারা একত্রিত করা হয়। তারপরে, ব্রাউজারের জাভাস্ক্রিপ্ট ইঞ্জিন জাভাস্ক্রিপ্ট ফাইল এবং ইনলাইন কোড লোড করে কিন্তু কোডটি অবিলম্বে চালায় না। এটি HTML এবং CSS লোডিং শেষ হওয়ার জন্য অপেক্ষা করে। একবার এটি হয়ে গেলে, কোডটি লেখার ক্রমে জাভাস্ক্রিপ্ট কার্যকর করা হয়। এর ফলে DOM জাভাস্ক্রিপ্ট কোড দ্বারা আপডেট হয় এবং ব্রাউজার দ্বারা রেন্ডার করা হয়। এখানে আদেশ গুরুত্বপূর্ণ. যদি জাভাস্ক্রিপ্ট HTML এবং CSS শেষ হওয়ার জন্য অপেক্ষা না করে, তবে এটি DOM উপাদানগুলি পরিবর্তন করতে সক্ষম হবে না।

ইভেন্ট লুপের একটি সহজ কাজ আছে — কল স্ট্যাক এবং কলব্যাক কিউ পরীক্ষা করা। কল স্ট্যাক খালি থাকলে, ইভেন্ট লুপ সারি থেকে প্রথম ইভেন্টটি নেবে এবং এটিকে কল স্ট্যাকের দিকে ঠেলে দেবে, যা কার্যকরভাবে এটি চালায়। এই ধরনের পুনরাবৃত্তিকে ইভেন্ট লুপে টিক বলা হয়। প্রতিটি ইভেন্ট শুধু একটি ফাংশন কলব্যাক।

localStorage এবং sessionStorage এর কিছু পার্থক আছে। সেশন স্টোরেজ সম্পত্তি আপনাকে একটি সেশন স্টোরেজ অবজেক্টে access করার অনুমতি দেয়। localStorage এর ডাটা গুলার কোন মেয়াদ নাই। আপনি সব সময় তা দেখতে পাবেন। লোকালস্টোরেজ এর ডাটা আপনাকে একটি স্থানীয় স্টোরেজ অবজেক্টে access করার অনুমতি দেয়। পার্থক্যটি হল লোকালস্টোরেজে থাকা data টির কোনও মেয়াদোত্তীর্ণের সময় নেই এবং সেশনসটোরেজে সংরক্ষিত data ব্রাউজার সেশন শেষে বা ব্রাউজারটি বন্ধ হয়ে গেলে মুছে যায়।
